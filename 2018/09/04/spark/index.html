RDD

RDD: resilient distributed dataset

transformation: map, filter, flatMap, sample, distinct, mapPartitions, …

  • return RDD
  • lazy calculated
  • will copy every referenced data around (do not refer big objects, save object reference to local variable)

action: reduce, fold, aggregate, take, top, first, takeOrdered, takeSample, foreach, collect, count

  • return local data structure
  • perform the calculation
  • do all the transformations every time (consider call persist to save the intermediate result)

set operations: union, intersection, subtract

aggregate:

fold/reduce: fold会严格按照顺序进行，计算的时候需要提供一个初始值，而reduce不会也不需要。reduce需要严格保证元素的操作的交换律和结合律，而fold没有这个要求。reduce是fold的一个特殊用例。

persist/unpersist: 保存数据，默认到JVM堆空间，可以指定StorageLevel

pair RDD

transformations:

  • for one pair RDD: reduceByKey, groupByKey, combineByKey, mapValues, flatMapValues, keys, values, sortByKey
  • for two pair RDDs: subtractByKey, join, rightOuterJoin, leftOuterJoin, cogroup

actions: countByKey, collectAsMap, lookup(key)

partition: partitionBy(partitioner)

很多操作需要做数据混洗（shuffle），如join/sortByKey/reduceByKey等，分区可以有效的减少数据的迁移。常常把数据量大的一个pair RDD分区，而只让另一个数据量小的pair RDD进行混洗，减少网络传输。

能利用分区信息提高性能的操作：cogroup, groupWith, join, leftOuterJoin, rightOuterJoin, groupByKey, reduceByKey, combineByKey, lookup

许多操作会自动生成一个Option[Partitioner]，执行后续步骤时spark会自动获取分区信息来提高效率。这些操作有cogroup, groupWith, join, leftOuterJoin, rightOuterJoin, groupByKey, reduceByKey, combineByKey, partitionBy, sort。父RDD有分区方式的话，mapValues, flatMapValues会保留分区信息。

page rank:

自定义分区方式

注入更多领域知识，进一步提升性能，如对于pageRank，可以认为同一个域名下的网页更可能互相链接，所以可以按照域名来进行分区。

python里面直接传一个函数作为分区函数，需要注意想让多个RDD使用相同的分区方式时，需要传入同一个分区函数

输入输出

文件格式：text, json, csv(如果允许包含\n，则需要读取全部数据再进行处理，有时会成为瓶颈), tsv(用tab分隔的csv), SequenceFile, Object, ProtoBuffer, HadoopFile

文件系统：本地文件 s3 hdfs

SequenceFile

由没有相对关系的键值对文件组成，有同步标记，可以定位到文件中的点，然后再与记录的边界对齐。这可以使spark从多个节点高效的并行读取数据。

Spark SQL读取文件格式：hive/json

数据库：jdbc connection, cassandra, hbase, elasticsearch

高级特性

accumulator:

在分布式操作中做一些统计，在驱动器程序上面可以访问value属性，工作节点无法访问value属性（只写）。可以扩展AccumulatorParam自定义累积器，需要保证操作满足交换律和结合律。

统计正确性（spark会重启失败的任务，或者抢占式地尝试在其他节点上面执行比较慢的任务–speculation）：

  • 行动操作中执行累积操作，可以保证只执行一次
  • 转换操作中无法保证正确性，因为可能因为缓存原因而没有计算，也可能计算多次；转换操作中的累积器常常只用于调试

broadcast:

对于在每个节点都要多次使用的只读的较大对象，提前发送到各个节点缓存，避免多次（多个RDD）发送，而且spark只对小对象发送有优化，可以避免性能问题。注意避免修改对象，否则将只对该工作节点生效

op based on partition: mapPartitions, mapPartitionsWithIndex, foreachPartitions

spark提供了一些基于分区操作的api，表示对该分区仅进行一次操作，而不是每次转换的时候都操作。适合连接外部数据库、读取外部数据源等初始化操作。
调用的时候，得到的是整个分区的元素的迭代器。

pipe to communicate with other application: rdd.pipe(Seq(SparkFiles.get("finddistance.R"), ","))

stats of rdd: 通过每次计算一个元素的方式出来。可以用来根据统计信息移除异常数据，此时注意需要缓存rdd，因为会使用多次。

在集群中运行

  • 集群管理器：spark独立集群管理器、YARN、mesos
  • 集群运行模式：client(驱动器在本地运行), cluster（驱动器在集群运行）
  • spark应用：驱动器节点、工作节点
  • 打包：借助maven shade plugin/sbt创建super jar
  • 依赖冲突：修改应用使用spark依赖相同的版本；使用shading方式（maven shade plugin）来自动修改包名
  • spark独立集群的高可用：使用zookeeper维护多个备用主节点
  • 配置应用的资源：–executor-memory, –total-executorcores

调试和调优

  • rdd.toDebugString: 得到步骤、任务等执行信息
  • Spark程序运行：将RDD组织为一个DAG，根据行动RDD反向查找并所有依赖RDD的树结构，生成物理执行计划（合并可能的操作为一个步骤以便进行流水线执行，使用缓存的RDD（截短操作序列）），在集群中调度并执行任务
  • 作业：特定的行动操作生成的步骤的集合，步骤有依赖关系，会以特定的顺序执行
  • 步骤：步骤会启动很多任务，每个任务在不同的数据分区上面做相同的事情
  • 任务的内部流程：从存储或者混洗结果中获取数据，执行filter map等操作，把输出写到一个数据混洗文件（外部存储）或者发挥驱动器程序
  • 任务日志：http://localhost:4040/jobs, work/目录
  • 性能指标：并行度、序列化格式、内存管理（指定各部分内存用量、缓存位置）、硬件供给（可以使用的memory大小、cpu数量、混洗的中间数据存储）

Spark SQL

从各种结构化数据源读取数据：json hive parquet；对外提供jdbc/odbc服务(Beeline)；与RDD高度整合，可以实现join union等操作；支持自定义函数（UDF）

registerTempTable, printSchema输出表结构信息, Row对象，数据表缓存

性能调优：codegen, memory compress, memory batch size, compression codec

Spark Streaming

使用离散化流来表示，DStream(discretized stream)，按照时间窗口进行数据处理，可以滑动窗口来实现跨窗口的计算。DStream支持转换操作（大部分RDD的功能）和输出操作（结果写入到外部存储等）。checkpoint机制，自动重启机制可以支持7x24不间断工作。

接收器将接收到的数据复制到另一个执行器以保证容错性，DStream的重算能力可以保证数据再次被计算出来。但是不能从头计算，引入checkpoint机制，就可以只算一部分。

有状态的转换操作：

  • 基于滑动窗口：dstream.window dstream.reduceByKeyAndWindow
  • 跨批次维护状态：dstream.updateStateByKey

输出：saveAsTextFiles saveAsHadoopFiles, 在foreachRDD中自定义保存方式

输入源：ssc.fileStream从文件创建，kafkaUtils.createStream订阅某个主题，flume的推式接收器和拉式接收器；支持多数据源同时输入；分配的核心至少为#接收器 + 1

不间断运行：

  • 驱动器程序：配置使用checkpoint，编写错误恢复代码，--supervise参数让机器自动重启驱动器程序，--deploy-mode cluster保证驱动器在集群中运行，配置集群的容错性
  • 工作节点容错：使用与Spark同样的方式，有多个数据备份，可以随时重算
  • 接收器容错：使用可靠的数据源（hadoopFiles）保证容错；使用不可靠数据源Kafka flume twitter时，如果接收器失败会导致部分数据丢失
  • 处理保证：使用事务操作写入外部系统，设计幂等更新操作

性能：批次、窗口大小、并行度、垃圾回收和内存使用
